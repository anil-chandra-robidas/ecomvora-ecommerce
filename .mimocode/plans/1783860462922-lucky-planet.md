# EcomVora — Food & Beverages Ecommerce Website

## Overview
A full-featured, eye-catching ecommerce website for Food & Beverages built with **Next.js 14 (App Router)** + **Tailwind CSS**. Brand palette: **orange + black**. Store name: **EcomVora**.

---

## Tech Stack

| Layer | Choice | Rationale |
|-------|--------|-----------|
| Framework | Next.js 14 (App Router) | SSR, SEO, file-based routing |
| Styling | Tailwind CSS 3.4 | Utility-first, rapid UI |
| Language | TypeScript | Type safety |
| Database | SQLite via Prisma | Zero-config, easy setup |
| ORM | Prisma 5 | Type-safe DB queries |
| Auth | NextAuth.js v5 | Google/email login |
| Payments | Stripe (test mode) | Industry standard |
| Icons | Lucide React | Lightweight, consistent |
| Animations | Framer Motion | Smooth, performant |
| State | React Context (cart) | No extra deps for cart |

## Database Schema (Prisma)

```prisma
model User {
  id            String    @id @default(cuid())
  name          String?
  email         String    @unique
  emailVerified DateTime?
  password      String?
  image         String?
  role          String    @default("customer") // "customer" | "admin"
  orders        Order[]
  reviews       Review[]
  accounts      Account[]
  sessions      Session[]
}

model Product {
  id          String    @id @default(cuid())
  name        String
  slug        String    @unique
  description String
  price       Float
  image       String
  category    String    // "fruits", "vegetables", "beverages", "snacks", "dairy", "bakery"
  stock       Int       @default(0)
  featured    Boolean   @default(false)
  createdAt   DateTime  @default(now())
  updatedAt   DateTime  @updatedAt
  orderItems  OrderItem[]
  reviews     Review[]
}

model Order {
  id            String      @id @default(cuid())
  userId        String
  user          User        @relation(fields: [userId], references: [id])
  items         OrderItem[]
  total         Float
  status        String      @default("pending") // pending, processing, shipped, delivered
  shippingName  String
  shippingEmail String
  shippingAddr  String
  paymentIntentId String?
  createdAt     DateTime    @default(now())
}

model OrderItem {
  id        String  @id @default(cuid())
  orderId   String
  order     Order   @relation(fields: [orderId], references: [id])
  productId String
  product   Product @relation(fields: [productId], references: [id])
  quantity  Int
  price     Float
}

model Review {
  id        String  @id @default(cuid())
  userId    String
  user      User    @relation(fields: [userId], references: [id])
  productId String
  product   Product @relation(fields: [productId], references: [id])
  rating    Int     // 1-5
  comment   String?
  createdAt DateTime @default(default(now()))
}
```

## Implementation Steps

### Step 1: Project Scaffold
- `npx create-next-app@latest ecomvora --typescript --tailwind --app --src-dir`
- Install deps: `prisma`, `@prisma/client`, `next-auth`, `stripe`, `framer-motion`, `lucide-react`, `bcryptjs`, `zod`

### Step 2: Tailwind Config + Global Styles
- Configure orange/black color palette in `tailwind.config.ts`
- Set up `globals.css` with custom scrollbar, animations

### Step 3: Database + Auth
- Set up Prisma schema, seed with 20+ food/beverage products
- Configure NextAuth with credentials + Google providers

### Step 4: Layout (Navbar + Footer)
- Build responsive Navbar with logo, search bar, cart icon, auth links
- Build Footer with newsletter signup, social links

### Step 5: Homepage
- Animated hero with gradient orange-to-black background
- Featured categories grid (Fruits, Beverages, Snacks, etc.)
- Trending products carousel
- Newsletter CTA section

### Step 6: Product Pages
- Product grid with filter sidebar (category, price range, search)
- Product detail page with image gallery, reviews, add-to-cart

### Step 7: Cart + Checkout
- Cart page with quantity controls, remove items, order summary
- Checkout with shipping form + Stripe payment

### Step 8: Admin Dashboard
- Protected admin layout with sidebar
- Dashboard with stats cards (orders, revenue, users)
- Product CRUD (create, edit, delete with image upload)
- Order management (view, update status)

### Step 9: Polish
- Framer Motion animations on page transitions, card hovers
- Loading states, empty states, error handling
- Responsive design (mobile-first)
- SEO metadata on all pages
