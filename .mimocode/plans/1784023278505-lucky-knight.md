# Stripe Checkout Integration Plan

## Overview
Replace the fake payment form with real Stripe Checkout (hosted payment page). User fills shipping info -> clicks Pay -> redirected to Stripe -> pays -> returns to success page.

## Flow
```
Checkout form -> POST /api/checkout -> creates Order (pending) + Stripe Session -> returns session URL
User redirected to Stripe -> enters card -> pays
Stripe redirects to /checkout/success?session_id=cs_xxx
Stripe webhook -> POST /api/webhooks/stripe -> verifies signature -> updates Order status to "paid"
Success page fetches order by session_id -> displays real order details
```

## Prerequisites (user action)
1. Sign up at https://dashboard.stripe.com
2. Get test API keys from Developers -> API keys
3. Add to Vercel env vars: `STRIPE_SECRET_KEY` (sk_test_...), `NEXT_PUBLIC_BASE_URL` (https://ecomvora-ecommerce.vercel.app)
4. For webhooks locally: `npm i -g stripe && stripe listen --forward-to localhost:3000/api/webhooks/stripe`

## Files to Create (3)

### 1. `src/lib/stripe.ts` ŌĆö Stripe client singleton
```ts
import Stripe from "stripe";
export const stripe = new Stripe(process.env.STRIPE_SECRET_KEY!);
```

### 2. `src/app/api/checkout/route.ts` ŌĆö Create checkout session + order
- Receives: items (productId, quantity), shippingName, shippingEmail, shippingAddr
- Server-side: fetches product prices from DB, calculates total + 8% tax
- Creates Order in DB (status: "pending")
- Creates Stripe Checkout Session with line items from DB prices
- Returns `{ url: session.url }`

### 3. `src/app/api/webhooks/stripe/route.ts` ŌĆö Handle payment confirmation
- Verifies Stripe signature using `STRIPE_WEBHOOK_SECRET`
- On `checkout.session.completed`: finds Order by `paymentIntentId` (stores session ID), updates status to `"paid"`
- Must use `request.text()` for raw body (not `request.json()`)
- Must have `export const dynamic = "force-dynamic"`
- Always returns 200 (Stripe retries on non-200)

## Files to Modify (4)

### 4. `src/app/checkout/page.tsx` ŌĆö Remove card inputs, redirect to Stripe
- Remove: card number, expiry, CVC form fields and the "Payment Details" section
- Remove: fake `setTimeout` payment simulation
- On submit: POST to `/api/checkout` (not `/api/orders`), then `window.location.href = session.url`
- Remove demo disclaimer text
- Keep: shipping form (name, email, address, city, zip), order summary sidebar

### 5. `src/app/checkout/success/page.tsx` ŌĆö Show real order details
- Add `"use client"` directive
- Read `session_id` from URL search params
- Fetch order from a lookup endpoint using session_id
- Display real order ID, total, and status
- Fallback to generic message if session_id missing

### 6. `src/types/index.ts` ŌĆö Update Order type
- Add `paymentIntentId?: string` to Order interface
- Add `"paid"` to status union type

### 7. `.env.example` ŌĆö Add Stripe env vars
```
STRIPE_SECRET_KEY=sk_test_...
STRIPE_WEBHOOK_SECRET=whsec_...
NEXT_PUBLIC_BASE_URL=http://localhost:3000
```

## Key Details
- Reuses existing `Order.paymentIntentId` field (prisma/schema.prisma:88) to store Stripe Session ID
- Tax calculated server-side (8%), not trusted from client
- Webhook endpoint needs `export const dynamic = "force-dynamic"` for Vercel
- `stripe` package already in package.json ŌĆö just needs `npm install`
- No Prisma schema changes needed
